[ 필수 기능 ]
요구사항 1 : 메인 메뉴판
 - 메뉴 선택 시 상세 메뉴화면 이동
 - 잘못된 번호 선택 시 예외처리
 - 프로그램 종료를 위한 번호 정의

 Lv.1
 - 프로그램 실행 시 메뉴판 번호 보여줌(대분류)
 - 대분류 메뉴를 선택하면 세부 메뉴들을 보여줌
 - 반복문으로 메뉴 선택을 유지함.

 Lv.2
 - 필요한 클래스들을 설계합니다.(버거, 아이스크림, 음료, 맥주, 주문, 공통 등)
 - 클래스들의 **`프로퍼티`**와 **`메소드`**를 정의합니다
     - 예를 들어 아래 이미지처럼 클래스 다이어그램을 그려봅시다.
     - 햄버거는 이름, 가격 같은 프로퍼티와 정보를 출력하는 메소드가 있을 수 있죠?
 - **`Lv1`**에서 작성한 로직을 메소드로 만듭니다.

Lv.4
- 문자 입력, 없는 숫자 입력 시 예외 처리로 다시 입력할 수 있도록 바꾸기 - v
- 잔액과 가격 비교해서 구매 가능한지 확인 - 캐릭터 만들기
 => object로 소비자, 키오스크 만들고, 키오스크는 메뉴 선택이랑 가격 계산
- 오류별 다른 문구 출력

Lv.5
- 특정 작업 종료 후 3초 뒤 다른 작업 수행
- 결제 시간이 특정 시간대면 결제 불가
- 결제 완료 시 시간 띄움
- 프로그램 종료 시마다 5초마다 현재 주문 대기수 실시간 출력

Lv.보완
- selectNum 함수 이름 변경, 내용 중 종료하는 기능 이사 고민
- 변수 이름들 어떻게 바꿀지 고민
- 함수들도 모아서 따로 클래스 만들지 고려
- WrongMenu 삭제 고려
- -1 입력 시 그냥 종료됨(-1을 종료로 받기 때문)
